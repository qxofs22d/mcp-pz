## Модуль: Хранилище расчётных данных (Data Core)

### 1. Спецификация ТЗ

**Название и место в архитектуре**  
MCP-сервер для управления параметрами, формулами и выполнения скриптов.

**Цель (зачем нужен)**  
- Централизованное хранение всех расчётных показателей и формул.  
- Возможность изменения данных и пересчёта через скрипты.

**Входные данные**  
Инструменты: `get_parameter`, `set_parameter`, `list_parameters`, `add_formula`, `execute_formula`, `run_script`, `export_data`, `import_data`.

**Выходные данные**  
- Запрошенные данные или статус операции.

**Основная логика**  
- Данные хранятся в JSON-файле `data/calculations.json`.  
- При изменении параметра помечать зависимые формулы как устаревшие.  
- Запуск скриптов в подпроцессе с таймаутом и передачей данных через файл.

**Нефункциональные требования**  
- Безопасное выполнение скриптов (изоляция, таймаут).
- Атомарное сохранение.

### 2. Стратегия реализации

**Исполнитель**  
MCP-сервер на Python с использованием `subprocess`, `asteval`.

**Критерии успеха**  
- Все операции с данными выполняются корректно.
- Скрипты запускаются и обновляют данные.

### 3. Критерии приемки
- [ ] Создание и изменение параметров.
- [ ] Добавление и вычисление формул.
- [ ] Запуск внешнего Python-скрипта и обновление данных.
- [ ] Экспорт/импорт данных в JSON.
