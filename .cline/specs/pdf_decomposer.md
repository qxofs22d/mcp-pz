## Модуль: Декомпозиция PDF

### 1. Спецификация ТЗ

**Название и место в архитектуре**  
Модуль декомпозиции PDF — MCP-сервер, анализирующий PDF-файлы и возвращающий структурированное представление (разделы, таблицы, формулы).

**Цель (зачем нужен)**  
- Извлечь иерархию разделов, текст, таблицы, формулы и названия чертежей из PDF.  
- Подготовить данные для индексации и для выделения расчётных блоков.

**Входные данные**  
- `decompose_pdf(pdf_path)` — путь к PDF-файлу (в папке проекта).

**Выходные данные**  
- JSON-объект с полями: `sections`, `standalone_tables`, `formulas`, `drawings` (см. пример в спецификации).

**Основная логика**  
1. Извлечение текста и таблиц с помощью PyMuPDF и pdfplumber.
2. Построение структуры разделов (по оглавлению или эвристикам).
3. Классификация блоков (текст, таблица, формула) по эвристикам.
4. Поиск названий чертежей (ключевые слова).
5. Сохранение результата в JSON-файл (кэш).

**Обработка ошибок**  
- При ошибке чтения PDF возвращать код ошибки и описание.

**Нефункциональные требования**  
- Время обработки до 3 мин на 100 стр.
- Кэширование результата.

### 2. Стратегия реализации

**Исполнитель**  
MCP-сервер на Python.

**Критерии успеха**  
- Для тестового PDF корректно извлекается структура и блоки.

### 3. Критерии приемки
- [ ] Обработка PDF с оглавлением даёт правильную иерархию разделов.
- [ ] Таблицы извлекаются в структурированном виде.
- [ ] Формулы (содержащие "=") идентифицируются.
- [ ] Названия чертежей находятся.
