# Спецификация модуля: LLM Gateway

## Назначение
Предоставляет доступ к внешним LLM (через OpenRouter) для экспертного режима, когда требуется глубокое объяснение или анализ.

## MCP-инструменты

### `ask(prompt, model=None, temperature=0.7, max_tokens=2000)`
Отправляет запрос к указанной модели и возвращает ответ.

- **prompt** (string, обязательный) — текст запроса
- **model** (string, опционально) — модель OpenRouter (по умолчанию из конфига)
- **temperature** (number, опционально) — температура генерации (0.0-2.0)
- **max_tokens** (integer, опционально) — максимум токенов в ответе

### `ask_with_context(prompt, context, model=None, temperature=0.7, max_tokens=2000)`
Отправляет запрос с дополнительным контекстом (например, история диалога, данные проекта).

- **context** (string) — контекстная информация
- остальные параметры как в `ask`

### `list_models()`
Возвращает список доступных моделей OpenRouter (популярные бесплатные и платные). Включает:
- `deepseek/deepseek-r1-0528:free` (бесплатная)
- `mistralai/mistral-7b-instruct:free` (бесплатная)
- `google/gemini-2.0-flash-exp:free` (бесплатная)
- `anthropic/claude-3-haiku-20240307` (платная)
- `openai/gpt-4o-2024-08-06` (платная)

### `set_default_model(model)`
Устанавливает модель по умолчанию для последующих запросов.

## Конфигурация
- API-ключ OpenRouter читается из переменной окружения `OPENROUTER_API_KEY` или из файла `.env` в корне проекта
- Модель по умолчанию: `deepseek/deepseek-r1-0528:free`
- Базовый URL: `https://openrouter.ai/api/v1`
- Таймаут запроса: 60 секунд
- Максимальное количество попыток при ошибке: 3

## Внутренняя структура
- **OpenRouterClient** — класс для работы с API (асинхронные запросы)
- **Config** — загрузка настроек из .env
- **ModelManager** — управление списком моделей и моделью по умолчанию

## Зависимости
```
aiohttp>=3.9.0
python-dotenv>=1.0.0
mcp>=1.0.0
```

## Логирование
Все запросы и ответы должны логироваться через модуль Logger:
- вызов `tools/call` к серверу Logger с типом "expert"
- запись промпта, модели, температуры и длины ответа

## Обработка ошибок
- Таймауты: возвращать понятное сообщение пользователю
- Ошибки API (неверный ключ, превышение лимита): логировать и предлагать проверить ключ
- Retry при временных сбоях сети (до 3 раз)

## Пример использования в промпте оркестратора
```
Если пользователь спрашивает "объясни подробно" или "почему", 
используй LLM Gateway:
1. Собери контекст (последние сообщения, данные проекта)
2. Вызови ask_with_context с моделью по умолчанию
3. Верни ответ с пометкой [Expert]
```

- [ ] Сервер запускается и регистрирует все инструменты
- [ ] Запросы к OpenRouter выполняются успешно (с реальным ключом)

- [ ] Логирование запросов через Logger работает
