## Модуль: Векторное хранилище (RAG-индекс)

### 1. Спецификация ТЗ

**Название и место в архитектуре**  
MCP-сервер для семантического индексирования и поиска по текстам PDF.

**Цель (зачем нужен)**  
- Индексировать чанки текста (500 токенов) из PDF.  
- Обеспечивать поиск релевантных фрагментов по запросу.

**Входные данные**  
- `add_document_chunks(document_name, chunks)` — добавить чанки документа.
- `search(query, top_k=5, filter_documents=None)` — поиск.

**Выходные данные**  
- Список чанков с метаданными и оценкой релевантности.

**Основная логика**  
- Использование sentence-transformers для эмбеддингов, ChromaDB для хранения и поиска.
- Разбивка на чанки фиксированного размера (500 токенов) с перекрытием.

**Нефункциональные требования**  
- Поиск < 1 сек.
- Индекс сохраняется на диске.

### 2. Стратегия реализации

**Исполнитель**  
MCP-сервер на Python с библиотеками sentence-transformers, chromadb.

**Критерии успеха**  
- Добавление документов и поиск работают корректно.

### 3. Критерии приемки
- [ ] Индексация текстового файла.
- [ ] Поиск по запросу возвращает релевантные чанки.
- [ ] Фильтрация по документам работает.
