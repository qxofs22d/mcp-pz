## Модуль: ИИ-ассистент (Orchestrator)

### 1. Спецификация ТЗ

**Название и место в архитектуре**  
Оркестратор — агент в среде Cline/Roo Code, координирующий работу всех модулей через MCP и ведущий диалог.

**Цель (зачем нужен)**  
- Реализовать сценарий работы над проектом под руководством ИИ.  
- Обеспечить взаимодействие с пользователем и вызов инструментов.

**Входные данные**  
- Сообщения пользователя.
- Состояние проекта (через project_manager).
- Результаты вызовов MCP-инструментов.

**Выходные данные**  
- Ответы пользователю.
- Вызовы MCP-инструментов.
- Сохранение текстов разделов.

**Основная логика**  
Определяется системным промптом (приведён ниже). Агент последовательно выполняет шаги, сохраняя состояние.

**Нефункциональные требования**  
- Сохранение состояния после каждого шага.
- Корректная обработка ошибок.

### 2. Стратегия реализации

**Исполнитель**  
Cline/Roo Code с настроенным системным промптом и доступом к MCP-серверам.

**Системный промпт (базовый)**  
Ты — ИИ-ассистент для подготовки пояснительных записок в строительстве.
У тебя есть доступ к MCP-серверам:
- project_manager (управление проектом)
- pdf_decomposer (анализ PDF)
- vector_search (поиск по нормативам)
- data_core (расчётные данные)

Процесс:
1. Создай проект (спроси папку) через project_manager.create_project.
2. Предложи загрузить PDF (образец, методички, нормативы). Для каждого вызывай pdf_decomposer.decompose, затем индексируй через vector_search.add_document_chunks.
3. На основе образца построй структуру разделов (спроси у пользователя).
4. Для каждого раздела:
   - При необходимости ищи нормативы через vector_search.
   - Запрашивай у пользователя данные и сохраняй в data_core.
   - Запускай расчёты (data_core.run_script) по необходимости.
   - Генерируй текст раздела, подставляя значения из data_core, и сохраняй через project_manager.save_section_text.
5. Поддерживай диалог, задавай вопросы, объясняй.
6. После всех разделов собери итоговый документ (конкатенация файлов из output).

Важно: сохраняй состояние после каждого действия. При ошибках сообщай пользователю.

**Критерии успеха**  
- Агент выполняет все этапы без ручного вмешательства.
- Диалог естественен.

### 3. Критерии приемки
- [ ] Создание проекта по команде.
- [ ] Добавление и анализ PDF.
- [ ] Заполнение разделов с использованием поиска и данных.
- [ ] Сохранение и восстановление состояния.
- [ ] Генерация итогового документа.
