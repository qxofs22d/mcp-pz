# План реализации модулей

Очередность разработки (каждый следующий модуль опирается на предыдущий):

1. **Logger** — модуль для логирования диалога в JSONL. Нужен для сохранения всех обсуждений с самого начала.
2. **Project Manager** — управление проектом, файлами, состоянием.
3. **Vector Storage** — индексация и поиск по тексту (используется для диалогов и документов). Поддержка нескольких коллекций: `documents`, `dialogue`, `system`.
4. **Data Core** — хранение параметров, формул, запуск скриптов.
5. **PDF Decomposer** — анализ PDF, извлечение структуры.
6. **LLM Gateway** — доступ к внешним LLM через OpenRouter (для экспертного режима).

После каждого модуля — демонстрация работоспособности (тестовый запуск в Cline). После реализации Vector Storage доработай его для поддержки коллекций. После реализации LLM Gateway настрой его на использование ключа из `.env`.
