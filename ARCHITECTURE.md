# Архитектура системы MCP-PZ

## Терминология

### 🛠️ Платформа (Platform)
**Путь:** `d:\dev\mcp-pz\`

Это **инструмент автоматизации** — набор MCP-серверов и утилит для обработки документов и подготовки ПЗ.

```
d:\dev\mcp-pz\                    # ПЛАТФОРМА
├── servers\                      # MCP-серверы (ядро)
│   ├── logger\                   # Логирование
│   ├── project_manager\          # Управление рабочими проектами
│   ├── pdf_decomposer\           # Декомпозиция PDF
│   ├── vector_storage\           # Векторный поиск
│   └── data_core\                # Хранилище данных
├── data\                         # Данные платформы
│   └── registry.json             # Реестр проектов
├── requirements.txt              # Зависимости платформы
├── .clinerules                   # Правила для ИИ-ассистента
├── ARCHITECTURE.md               # Этот файл
├── PLATFORM_LOGIC.md             # Логика работы
└── CLINE_INTERFACE.md            # Настройка интерфейса Cline
```

**Назначение:** Предоставляет инструменты для работы с документами.

---

### 📁 Рабочий проект (Work Project)
**Путь:** `d:\Projects\{project_name}\`

Это **продукт работы** — конкретный проект ПЗ с исходными документами и результатами анализа.

```
d:\Projects\kr_foundation\        # РАБОЧИЙ ПРОЕКТ
├── .vscode\                      # Конфигурация VS Code
│   └── mcp.json                  # Конфигурация MCP-серверов
├── .clinerules                   # Специфика проекта (дисциплина, нормативы)
├── pdf\                          # Исходные документы
│   ├── Пример курсовой работы.pdf
│   └── Методические указания.pdf
├── output\                       # Результаты анализа
│   ├── summary.md                # Краткое содержание
│   ├── parameter_catalog.md      # Каталог параметров
│   ├── formula_catalog.md        # Каталог формул
│   └── missing_documents.md      # Недостающие документы
├── data\                         # Расчётные данные
│   └── calculations.json
├── vector_index\                 # Векторный индекс для поиска
└── state.json                    # Состояние проекта
```

**Назначение:** Хранит данные конкретного проекта ПЗ.

## Словарь терминов

| Термин | Английский | Описание | Пример |
|--------|------------|----------|--------|
| **Платформа** | Platform | Инструмент автоматизации | `d:\dev\mcp-pz` |
| **Рабочий проект** | Work Project | Конкретный проект ПЗ | `d:\Projects\kr_foundation` |
| **MCP-сервер** | MCP Server | Модуль платформы | `servers/pdf_decomposer` |
| **Документ** | Document | PDF-файл в рабочем проекте | `pdf/Курсовая работа.pdf` |
| **Каталог** | Catalog | Сгенерированный справочник | `output/parameter_catalog.md` |
| **Параметр** | Parameter | Расчётная величина | `γ = 19,5 кН/м³` |
| **Формула** | Formula | Расчётное выражение | `S = β × Σ(σzp × h / E)` |

## MCP-серверы

| Сервер | Назначение | Основные инструменты |
|--------|------------|---------------------|
| **Logger** | Логирование действий | `log` |
| **Project Manager** | Управление проектами | `init_project`, `add_pdf`, `get_state` |
| **PDF Decomposer** | Анализ PDF | `decompose_pdf`, `extract_tables`, `extract_formulas` |
| **Vector Storage** | Семантический поиск | `search`, `add_document_chunks`, `split_text` |
| **Data Core** | Хранилище данных | `get_parameter`, `set_parameter`, `add_formula`, `execute_formula` |

---

## Реестр проектов

**Файл:** `d:\dev\mcp-pz\data\registry.json`

Централизованный список всех проектов ПЗ. Используется Project Manager для:
- Инициализации новых проектов
- Поиска существующих проектов
- Отслеживания статуса проектов

---

## Временные данные проекта

| Папка | Назначение | Удалять? |
|-------|------------|----------|
| `pdf_cache/` | Кэш декомпозиции PDF | Можно (пересоздастся) |
| `vector_index/` | Индекс векторного поиска | Можно (перестроится) |

Эти данные **не являются продуктами** и могут быть удалены или перегенерированы.

---

## Рекомендации

1. **Чёткое разделение:** Платформа — это инструмент, Рабочий проект — это данные
2. **Git:** Платформа под версионным контролем, Рабочие проекты — опционально
3. **Резервное копирование:** Рабочие проекты важнее, Платформу можно восстановить из Git
4. **`.clinerules`:** В проекте описывает специфику (дисциплина, нормативы), в платформе — общие правила

---
